## Pitfalls

### P001: Prefer LiteralPath For Exact Paths

- Category: `path`, `powershell`
- Symptom: Paths with spaces, brackets, wildcard characters, or non-ASCII segments are misread or unexpectedly expanded.
- Cause: `-Path` treats wildcards and provider syntax specially.
- Prefer: Use `-LiteralPath` for known filesystem targets: `Get-ChildItem -LiteralPath 'D:\path\file.txt'`.
- Avoid: Passing exact user/workspace paths through `-Path` unless wildcard expansion is intended.
- Count: 1
- Source: Repeated Codex Windows filesystem operations.

### P002: Do Not Cross Shells For Destructive File Actions

- Category: `filesystem`, `sandbox`
- Symptom: A file list produced in PowerShell is passed to `cmd /c`, batch builtins, or another shell for deletion or moving.
- Cause: Quoting, wildcard, encoding, and path semantics change across shells.
- Prefer: Use one shell end to end, usually PowerShell native cmdlets with `-LiteralPath`.
- Avoid: `Get-ChildItem ... | ForEach-Object { cmd /c del ... }` and similar cross-shell deletion patterns.
- Count: 1
- Source: Windows safety rules in active Codex environment.

### P003: Validate Recursive Delete And Move Targets

- Category: `filesystem`, `sandbox`
- Symptom: Recursive delete or move could affect a broader path than intended.
- Cause: Computed paths, symlinks, relative paths, or missing variables can resolve outside the expected workspace.
- Prefer: Resolve the absolute target first and check it stays under the intended directory before `Remove-Item -Recurse` or recursive `Move-Item`.
- Avoid: Running recursive operations against computed strings that have not been resolved and compared.
- Count: 1
- Source: Windows safety rules in active Codex environment.

### P004: Escape For Every Layer

- Category: `quoting`, `path`, `powershell`
- Symptom: Backslashes disappear, regexes over-match, JSON command payloads become invalid, or Windows paths are interpreted incorrectly.
- Cause: A command often passes through JSON, PowerShell, regex, and Windows path parsing; each layer has its own escaping rules.
- Prefer: Reduce layers; use single quotes in PowerShell for literal strings; use `-LiteralPath`; use forward slashes inside Git SSH options when accepted; test complex patterns with read-only commands first.
- Avoid: Building commands that require mentally tracking JSON escaping plus PowerShell escaping plus regex escaping plus path escaping in one string.
- Count: 1
- Source: Recent session commands with `core.sshCommand`, `Where-Object -match`, and Windows paths.

### P005: PowerShell Web Cmdlets May Be Less Reliable Than Curl Or Python

- Category: `network`, `powershell`
- Symptom: `Invoke-RestMethod` failed with "The underlying connection was closed" while querying GitHub API.
- Cause: PowerShell/.NET TLS or HTTP stack behavior can differ from other clients in the same environment.
- Prefer: Try `curl.exe` or Python `urllib.request` with a `User-Agent` header when PowerShell web cmdlets fail.
- Avoid: Repeating `Invoke-RestMethod` retries without changing the client stack.
- Count: 1
- Source: 2026-06-28 Codex session installing GitHub CLI.

### P006: Env: Enumeration Can Fail On Duplicate Keys

- Category: `powershell`, `environment`
- Symptom: `Get-ChildItem Env:` failed with "An item with the same key has already been added."
- Cause: PowerShell environment provider can choke on duplicate or conflicting environment entries.
- Prefer: Query specific variables with `[Environment]::GetEnvironmentVariable('NAME')`, or inspect a narrowed set through another method.
- Avoid: Assuming `Get-ChildItem Env:` is always safe for broad environment dumps.
- Count: 1
- Source: 2026-06-28 Codex session checking GitHub environment variables.

### P007: Use Full Paths Until PATH Refreshes

- Category: `process`, `path`
- Symptom: A newly installed executable is unavailable in the current Codex shell even though installation succeeded.
- Cause: The running shell process does not refresh `PATH` automatically after installers or profile changes.
- Prefer: Use the discovered full executable path for the rest of the session, or start a fresh shell if needed.
- Avoid: Treating `command not found` after installation as proof the install failed.
- Count: 1
- Source: Semantic-alignment records note current shell PATH had not refreshed after installation.

### P008: Use SSH-Over-443 Explicitly For GitHub When Needed

- Category: `git-ssh`, `network`
- Symptom: GitHub push/ls-remote needs to work from a restrictive network.
- Cause: Standard SSH port 22 may be blocked while `ssh.github.com:443` works.
- Prefer: `git -c core.sshCommand="ssh -i C:/Users/Administrator/.ssh/key -p 443 -o HostName=ssh.github.com -o StrictHostKeyChecking=accept-new" ...`.
- Avoid: Repeated plain `git push` attempts when network policy is the likely blocker.
- Count: 1
- Source: 2026-07-04 semantic-alignment GitHub publish session.

### P009: Expect LF/CRLF Warnings

- Category: `line-endings`, `git`
- Symptom: Git reports `LF will be replaced by CRLF the next time Git touches it`.
- Cause: Windows checkout settings and repository text attributes can normalize line endings.
- Prefer: Treat warnings as signal to inspect `.gitattributes` or Git config only when the task cares about line endings.
- Avoid: Reformatting unrelated files just to silence line-ending warnings.
- Count: 1
- Source: 2026-07-04 semantic-alignment examples diff.

### P010: Mojibake In Session Indexes Is Evidence To Handle Encoding Carefully

- Category: `encoding`
- Symptom: Chinese thread names appear as mojibake in `session_index.jsonl` output.
- Cause: Console/codepage/rendering mismatch can garble display even when JSON structure is readable.
- Prefer: Preserve raw files, use UTF-8-aware tooling, and avoid rewriting session metadata unless encoding is explicitly verified.
- Avoid: Concluding that the underlying data is corrupt from console display alone.
- Count: 1
- Source: 2026-07-04 `session_index.jsonl` displayed Chinese titles as mojibake.

### P011: Hide Background Helper Windows

- Category: `process`, `powershell`
- Symptom: Starting background helpers can open visible windows in the user's desktop.
- Cause: `Start-Process` defaults may create visible interactive windows.
- Prefer: Use `Start-Process ... -WindowStyle Hidden` for background services or helpers unless the user explicitly asked to see it.
- Avoid: Launching visible background processes that the user does not need to control.
- Count: 1
- Source: Active Codex Windows safety rules.

### P012: Keep Approval Prefix Rules Narrow

- Category: `sandbox`, `process`
- Symptom: An escalation request needs repeatability without granting overly broad command execution.
- Cause: Broad prefixes such as `python` or generic shells can authorize unrelated future commands.
- Prefer: Use the shortest safe prefix that captures the repeated operation, such as `["git","-C","D:\work\repo","push"]` or a specific script path.
- Avoid: Prefix rules for arbitrary interpreters, destructive commands, or whole shells.
- Count: 1
- Source: Active Codex escalation policy and recent Git push approvals.

### P013: Do Not Attach Prefix Rules To Heredoc/Herestring Commands

- Category: `sandbox`, `quoting`
- Symptom: A command uses a heredoc/herestring and also tries to request reusable escalation.
- Cause: Heredocs/herestrings hide arbitrary script bodies behind a broad command prefix.
- Prefer: For reusable approvals, call a concrete script file or a narrow executable command. For one-off heredocs, request escalation without `prefix_rule`.
- Avoid: `@' ... '@ | python -` with a reusable `prefix_rule`.
- Count: 1
- Source: Active Codex escalation policy.

### P014: New-Item may not support LiteralPath

- Category: `powershell`
- Symptom: New-Item -ItemType Directory -Force -LiteralPath failed with a parameter-not-found error.
- Cause: The available Windows PowerShell version exposes -Path for New-Item but not -LiteralPath, even though other cmdlets support -LiteralPath.
- Prefer: Use [System.IO.Directory]::CreateDirectory($absolutePath) for exact directory creation, or verify New-Item parameters before using -LiteralPath.
- Avoid: Assuming every filesystem cmdlet accepts -LiteralPath.
- Count: 1
- Source: 2026-07-04 codex-windows-pitfalls install sync attempt; recorded 2026-07-04.
